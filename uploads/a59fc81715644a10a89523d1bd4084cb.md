## 1. 项目概述

### 1.1 项目背景

小团队需要一个挂在云端的协同软件，实现团队管理和项目管理。

### 1.2 核心需求

| 需求 | 描述 |
|------|------|
| 人员管理 | 用户注册、登录、角色、权限管理 |
| 项目管理 | 项目创建、分发、任务认领 |
| 文档管理 | Cell 结构文档、协作编辑、实时同步、云端存储+下载 |
| 远程访问 | IP + 端口 + 身份验证 |

### 1.3 技术约束

| 约束 | 要求 |
|------|------|
| 技术栈 | HTML + JS + Python + MySQL |
| 部署方式 | 公有云 |
| 隐私要求 | 防爬虫，访问控制 |
| 协作模式 | 各自编辑，实时同步（基于 Yjs CRDT） |
| 文档编辑 | Cell 结构，不需要代码执行 |
| 版本管理 | Git（后期迭代，P1） |

---

## 2. 功能需求

### 2.1 人员管理（P0）

| 功能 | 描述 |
|------|------|
| 用户注册 | 邮箱/密码注册 |
| 用户登录 | Session-based 认证 |
| 角色管理 | 管理员、普通成员 |
| 权限控制 | 基于角色的访问控制（RBAC） |

### 2.2 项目管理（P0）

| 功能 | 描述 |
|------|------|
| 项目创建 | 创建项目，设定成员 |
| 项目分发 | 将项目分配给团队成员 |
| 任务认领 | 成员主动认领任务 |
| 任务管理 | 任务状态（待处理/进行中/已完成） |
| 文件管理 | 项目文件上传、下载 |

### 2.3 文档管理（P0）

| 功能 | 描述 |
|------|------|
| Cell 编辑器 | Cell 结构文档（代码 cell + 文本 cell） |
| 实时协作 | 多人同时编辑，自动同步 |
| 云端存储 | 文档保存在云端 MySQL |
| 下载渠道 | 支持导出/下载文档 |

### 2.4 远程访问（P0）

| 功能 | 描述 |
|------|------|
| 接入方式 | IP + 端口 + 身份验证 |
| 认证 | 登录即身份验证 |
| 防爬虫 | 请求频率限制、访问控制 |

### 2.5 Git 版本管理（P1，后期迭代）

| 功能 | 描述 |
|------|------|
| 文档版本控制 | 基于 Git 的版本历史 |
| 分支管理 | 文档分支与合并 |

## 3. 技术方案

### 3.1 技术选型

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | HTML/JS + CodeMirror + Quill + Yjs | Cell 结构编辑器 + CRDT 协作 |
| 后端 | Flask + Flask-SocketIO | 轻量 Python 框架 + 实时通信 |
| 数据库 | MySQL | 关系型数据存储 |
| 实时协作 | Yjs (CRDT) | 冲突解决与同步 |
| 部署 | Docker | 云端一键部署 |
